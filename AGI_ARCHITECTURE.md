# Архитектура Когнитивного ИИ-Агента (Magdalina)

Этот документ описывает архитектуру автономного, самосовершенствующегося ИИ-агента, построенную на принципах когнитивной психологии и нейробиологии человека.

## Основные концепции

### 1. Трехуровневая Когнитивная Модель

Архитектура разделена на три уровня обработки информации:

*   **Сознание (Consciousness / Main Loop):**
    *   **Функция:** Принятие решений "здесь и сейчас", фокус внимания, решение текущей задачи.
    *   **Механизм:** Главный цикл (Main Loop), в котором агент анализирует текущий контекст, выбирает навыки из дерева навыков, планирует действия и обращается к LLM для генерации ответов и команд.
    *   **Особенности:** Имеет ограниченный объем рабочей памяти (аналог кратковременной памяти человека).

*   **Подсознание (Subconscious / Background Workers):**
    *   **Функция:** Фоновая обработка информации, рефлексия, консолидация памяти, оценка действий.
    *   **Механизм:** Асинхронные фоновые процессы. Они анализируют логи действий сознания, выявляют паттерны успеха/неудачи, формируют "опыт" и обновляют долгосрочную память.
    *   **Особенности:** Именно здесь происходит "математический" расчет эмоций (поощрение/наказание) на основе метрик успешности.

*   **Бессознательное (Unconscious / Core Constraints & Instincts):**
    *   **Функция:** Базовые инстинкты, защитные механизмы, незыблемые правила (Safety), глубинные цели.
    *   **Механизм:** Жестко заданные системные промпты, фильтры безопасности и базовые эвристики, которые не могут быть изменены агентом самостоятельно.

### 2. Динамическая Память (Опыт)

Память агента имитирует человеческую:
*   **Сенсорная/Кратковременная (Рабочая):** Контекст текущего диалога или задачи.
*   **Долгосрочная (Эпизодическая и Семантическая):** Хранится в векторной базе данных (RAG) и графах знаний.
*   **Механизм перезаписи и угасания:**
    *   Записи в памяти имеют "вес" (значимость) и "эмоциональный окрас".
    *   Часто используемые и высокоэмоциональные воспоминания (важные уроки, критические ошибки) закрепляются.
    *   Редко используемые воспоминания со временем угасают (архивируются, сжимаются через summarization или удаляются).

### 3. Эмоции как Математический Процесс

Эмоции выступают в роли системы внутреннего подкрепления (Reinforcement):
*   **Удовольствие/Поощрение:** Возникает при успешном выполнении задачи, достижении подцелей, получении позитивного фидбека. Математически выражается в повышении веса связанных с действием путей и нейронов/навыков.
*   **Страх/Негатив:** Возникает при ошибках, исключениях, негативном фидбеке. Приводит к "избеганию" подобных паттернов в будущем.
*   **Интерес/Любопытство:** Стимулирует эксплорацию (поиск новых путей) при недостатке информации.

### 4. Иерархия Навыков (Skills Tree)

Агент не является монолитным. Он состоит из набора навыков (инструментов или субагентов):
*   Навыки организованы иерархически. Например: "Взаимодействие с ОС" -> "Работа с файлами", "Использование браузера".
*   "Программирование" — это отдельный продвинутый навык, который может опираться на базовые навыки работы с компьютером.
*   Сознание динамически подгружает (активирует) нужный навык (skillset) в зависимости от контекста задачи, экономя ресурсы и улучшая фокус.

## Технологический Стек

*   **Язык программирования:** Python
*   **Среда выполнения:** Docker & Виртуальные машины (изолированные песочницы для безопасного выполнения кода).
*   **Мозг (LLM):** API облачных моделей (например, OpenAI, Anthropic) для сложных рассуждений + возможно локальные модели для быстрых фоновых задач.
*   **Память:** Векторные БД (Qdrant / Chroma / Pinecone) для долгосрочной памяти, Redis для кратковременной и стейта.
*   **Интерфейс (Дашборд):** Telegram-бот для взаимодействия, постановки задач, получения отчетов и мониторинга "эмоционального" состояния агента.
